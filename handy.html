3 days Challenge - Handy Phone

Overview

Initiated by Tink Labs, the first hospitality technology company. Handy phone, is meant for bringing solutions for hospitality with all-in-one services designed specially for hotels and travelers. With Handy phone, guests can remote control in-room amenities, stay connected and explore destinations, and process hotel services.

In April 2018, I recevied a 3-day challenge to design a express check out feature for hotel guests. Guests may uses this feature to view and check out their bills in simple steps.


Scenario

A guest stayed a hotel providing handy for two nights, ordered two in-room dining orders, one Disneyland ticket by using handy phone during the stay.


Process

  • Identify Stakeholders
  • Identify Functional Requirements/ Non-functional Requirements
  • Case Studies
  • Paper Sketching
  • Wireframes

The Who

Travelers who qualify for express checkout if they are a registered user and have previously placed a successful order where your billing, shipping, email and payment information were validated. If user make changes to their primary billing or shipping details, user will not qualify for Express Checkout until the next time they place an order.


The What

Consider what users expect with using the function:
  • Fast
  • Directly leading to final steps in the checkout process (Review & Place Order)
  • Primary Billing Info (Shipping, billing, email, credit card) is automatically preload based on the previous purchases


Functional Requirement

  • User must able to select his payment method
  • User must able to add new payment method
  • User must able to review the order
  • User must able to review each order in detail
    (Date Time, Quantity, Price, Event detail)
  • User must able to pay everything at once, if thereâ€™s problems, they cannot complete express checkout
  • User shall able to put in discount code


3-Click Solution

Review Order > Payment > Complete
